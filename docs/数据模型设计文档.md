# 数据模型设计文档

包括各个数据库表的设计、状态字段说明等

## image_asset数据库表

## 数据库表

| 字段                    | 类型            | 说明                                 |
| ----------------------- | --------------- | ------------------------------------ |
| id                      | BIGINT UNSIGNED | 主键，雪花 ID 或自增 ID              |
| file_name               | VARCHAR(255)    | 系统生成文件名                       |
| original_file_name      | VARCHAR(255)    | 用户原始文件名                       |
| file_hash               | CHAR(64)        | 文件 hash                            |
| file_size               | BIGINT UNSIGNED | 文件大小，单位 byte                  |
| mime_type               | VARCHAR(100)    | MIME 类型                            |
| file_ext                | VARCHAR(20)     | 文件扩展名                           |
| width                   | INT UNSIGNED    | 图片宽度                             |
| height                  | INT UNSIGNED    | 图片高度                             |
| storage_path            | VARCHAR(500)    | 图片存储路径                         |
| thumbnail_path          | VARCHAR(500)    | 缩略图路径，可为空                   |
| image_status            | TINYINT         | 图片状态                             |
| vector_status           | TINYINT         | 向量状态                             |
| vector_updated_time     | DATETIME        | 向量完成更新时间，可为空             |
| processing_started_time | DATETIME        | 向量化开始时间，可为空               |
| retry_count             | INT UNSIGNED    | 重试次数，默认 0                     |
| fail_reason             | VARCHAR(1000)   | 最近失败原因，可为空                 |
| version                 | INT UNSIGNED    | 乐观锁版本号，用于并发状态更新默认 0 |
| created_time            | DATETIME        | 创建时间                             |
| updated_time            | DATETIME        | 更新时间                             |

- `id`与向量库中`vector_id`一一对应

- 缩略图后续也是由系统进行异步添加生成任务



### 状态字段说明

1. **image_status 库中图片状态**

| 值   | 状态    | 含义         | 是否可展示 | 是否可搜索           | 说明                             |
| ---- | ------- | ------------ | ---------- | -------------------- | -------------------------------- |
| 1    | NORMAL  | 正常图片资产 | 是         | 取决于 vector_status | 上传成功、文件存在、图片可用     |
| 2    | DELETED | 已删除       | 否         | 否                   | 逻辑删除，不物理删除数据库记录   |
| 3    | INVALID | 图片资产异常 | 否         | 否                   | 文件缺失、文件损坏、图片不可读取 |

2. **vector_status 向量化状态**

| 值   | 状态       | 含义         | 是否可搜索 | 说明                             |
| ---- | ---------- | ------------ | ---------- | -------------------------------- |
| 1    | PENDING    | 待向量化     | 否         | 图片已入库，但还没有生成向量     |
| 2    | PROCESSING | 向量化处理中 | 否         | 任务正在执行                     |
| 3    | READY      | 向量已可用   | 是         | 向量已成功写入向量数据库         |
| 4    | FAILED     | 向量化失败   | 否         | 达到重试上限，或遇到不可恢复错误 |

3. **fail_reason 向量化失败原因字段**

| fail_reason             | 含义                                     |
| ----------------------- | ---------------------------------------- |
| FILE_NOT_FOUND          | 图片文件不存在                           |
| IMAGE_DECODE_ERROR      | 图片解码失败                             |
| MODEL_SERVICE_TIMEOUT   | 模型服务超时                             |
| MODEL_SERVICE_ERROR     | 模型服务异常                             |
| VECTOR_DIM_MISMATCH     | 向量维度不匹配                           |
| VECTOR_DB_UPSERT_FAILED | 向量库写入失败                           |
| READY_UPDATE_FAILED     | 向量库写入成功，但 MySQL 更新 READY 失败 |
| RETRY_LIMIT_EXCEEDED    | 超过最大重试次数                         |

## 向量库表

| 字段         | 存放位置 | 说明                   |
| ------------ | -------- | ---------------------- |
| id           | 向量主键 | 使用 `image_asset.id`  |
| embedding    | 向量字段 | 图片 embedding         |
| created_time | payload  | 向量写入时间，用于排查 |

- 写入方式使用`upsert`

## 日志系统Log文件

| 日志文件          | 记录内容                                                     | 主要来源                                                     |
| ----------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| app.log           | 系统通用运行日志、启动日志、配置加载日志、普通业务日志       | 全部模块                                                     |
| error.log         | 异常日志、未捕获异常、业务失败堆栈                           | 全部模块                                                     |
| upload.log        | 图片上传入库链路日志                                         | image-asset、storage                                         |
| vectorization.log | 图片向量化任务日志                                           | vectorization、model-client、vector-index                    |
| search.log        | 搜索链路日志                                                 | search、vectorization、model-client、vector-index、image-asset |
| storage.log       | 文件保存、读取、删除、文件缺失、孤儿文件相关日志             | storage、image-asset                                         |
| model-client.log  | Python 模型服务调用日志                                      | model-client                                                 |
| vector-index.log  | 向量库 search、upsert、delete、collection 配置日志           | vector-index                                                 |
| event.log         | 待补偿事件记录日志，例如孤儿文件、孤儿向量、任务发布失败、PROCESSING 超时；MVP 只记录，不消费、不自动处理 | common EventPublisher / EventLog                             |
| slow.log          | 慢请求、慢搜索、慢模型调用、慢向量库调用、慢 DB 回表         | 全部模块                                                     |
| access.log        | HTTP 请求访问日志，可选                                      | Controller 层 / Web Filter                                   |

当前阶段调整为：

```text
event.log 启用，但只作为待补偿事件的结构化记录文件。
MVP 不设计补偿事件表，不做事件消费，不做事件状态流转，也不执行自动补偿。
EventPublisher / EventLog 后续可以扩展为落库、投递 MQ 或被补偿任务扫描。
```

因此，暂不使用的 log 为：

```text
slow.log
access.log
```



### 日志格式说明

每一个日志项都按照以下格式填写：

| 字段      | 说明                                                         |
| --------- | ------------------------------------------------------------ |
| timestamp | 日志时间                                                     |
| level     | 日志级别，例如 INFO、WARN、ERROR                             |
| traceId   | 链路追踪 ID，用于串联上传、向量化、搜索等完整链路            |
| requestId | HTTP 请求 ID，在请求进入后端时生成                           |
| module    | 模块名，例如 image-asset、vectorization、search、storage、model-client、vector-index |
| bizType   | 业务类型，例如 IMAGE_UPLOAD、VECTORIZATION、SEARCH、VECTOR_INDEX |
| bizId     | 业务 ID，统一使用 image_id；如果当前节点还没有 image_id，可为空 |
| eventName | 当前日志事件名，例如 UPLOAD_START、VECTOR_UPSERT_SUCCESS、SEARCH_FAILED |
| status    | 当前节点状态，例如 START、SUCCESS、FAILED、SKIPPED           |
| costMs    | 当前节点耗时，单位毫秒                                       |
| errorCode | 错误码，成功时为空                                           |
| message   | 日志说明，包含必要上下文信息                                 |

- `traceId`和`requestId`区分：

  | 字段      | 含义                  | 生成时机                                                 | 作用范围                 |
  | --------- | --------------------- | -------------------------------------------------------- | ------------------------ |
  | requestId | 一次 HTTP 请求的 ID   | 请求进入 Java 后端时，由 Filter / Interceptor 生成       | 只标识当前这一次接口请求 |
  | traceId   | 一条完整业务链路的 ID | 链路开始时生成，后续同步调用、异步任务、服务调用继续传递 | 串联完整业务流程         |

  - 后续发起异步任务（向量化、调用python模型）任务的时候，`requestId`可以由发起方重新设置，但`traceId`进行透传来标记完整流程链路
