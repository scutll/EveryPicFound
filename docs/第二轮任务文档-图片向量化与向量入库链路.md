# 第二轮任务文档 - 图片向量化与向量入库链路

## 1. 当前阶段目标

当前目标不是一次性实现完整搜索系统，也不是重新设计所有模块，而是补齐这条主链路：

```text
图片上传入库成功
→ VectorizationTaskPublisher.publish
→ ThreadPoolVectorizationTaskPublisher 提交本地线程池
→ ImageVectorizationProcessor.process
→ 查询 image_asset
→ 校验图片状态
→ 更新 vector_status = PROCESSING
→ 读取本地图片文件
→ 调用 Python 图片向量化接口
→ 校验 embedding
→ 写入向量库
→ 更新 vector_status = READY
```

本轮任务的核心目标是：

```text
让“图片已入库并发布向量化任务”
真正变成
“图片向量生成成功、写入向量库、MySQL 状态更新为 READY”
```

当前阶段只做“上传后图片入库向量化链路”，不把完整搜索链路混进 P0。

本轮 P0 只覆盖：

```text
图片上传成功后的异步向量化
图片文件读取
Python 图片向量化调用
向量库 upsert
image_asset 状态流转
基础失败处理
```

本轮暂不实现：

```text
以图搜图
以文搜图
图文联合搜图
SearchPipeline
QueryVectorizerSelector
VectorSearchClient.search
HybridFusionStrategy
复杂补偿扫描
完整健康检查体系
```

---

## 2. 本轮必要开发范围判断

本轮判断“必要开发类”的标准是：

```text
1. 主链路编译必须依赖
2. 主链路运行必须依赖
3. 状态流转必须依赖
4. 失败处理必须依赖
5. 后续搜索类不提前混入 P0
```

因此，本轮必须优先完成的模块是：

| 模块 | 本轮作用 | 结论 |
|---|---|---|
| `common` | 异步上下文、日志、指标、线程池、事件记录 | 必须补基础可用实现 |
| `image-asset` | 查询图片元数据、更新图片和向量状态 | 必须补查询和状态服务 |
| `storage` | 检查并读取本地图片文件 | 必须补 `exists/read` |
| `model-client` | Java 调 Python 图片向量化服务 | 必须新增图片向量化调用能力 |
| `vector-index` | 写入图片向量、解析 active collection | 必须新增 upsert 和 collection 配置能力 |
| `vectorization` | 发布任务、执行任务、失败处理 | 本轮核心模块 |

### 3.1.2 `ThreadPoolVectorizationTaskPublisher`

当前状态：已有壳，但不能停留在“假发布”。

下一步任务：

```text
1. 注入 ExecutorProvider
2. 注入 ImageVectorizationProcessor
3. 注入 ContextAwareTaskDecorator
4. 使用 ExecutorBizType.VECTORIZATION 获取线程池
5. 使用 ContextAwareTaskDecorator 包装任务
6. 在线程池中执行 processor.process(command)
7. 捕获 RejectedExecutionException
8. 返回 VectorizationPublishResult
9. 记录发布日志和发布指标
```

它不做：

```text
不查数据库
不读文件
不调 Python
不写向量库
不更新 READY
```

策略原因：Publisher 只负责“任务提交”，不能膨胀成业务处理 Service。后续切 MQ 时只替换 Publisher 实现。

## 3.2 common 基础组件

### 3.2.1 `ContextAwareTaskDecorator`

当前状态：类存在但需要补实现。

下一步任务：

```text
1. 在线程提交前读取 RequestContextHolder.get()
2. 在线程执行时 RequestContextHolder.set(context)
3. finally 中 RequestContextHolder.clear()
```

策略原因：上传请求和异步向量化任务不在同一个线程中执行。如果不透传上下文，`traceId` 会断掉，日志无法串联。

名词解释：`TaskDecorator` 是任务装饰器，用来在任务真正执行前后增加上下文设置、清理等横切逻辑。

## 5.3 `DefaultImageVectorizationProcessor`

当前状态：本轮最核心编排类。

依赖接口：

```text
ImageAssetQueryService
ImageAssetStatusService
ImageAssetStateChecker
FileStorageService
ModelVectorizationClient
ActiveCollectionResolver
VectorIndexClient
VectorizationRetryPolicy
VectorizationFailureHandler
LogService
MetricRecorder
EventPublisher
```

主流程：

```text
1. 接收 ImageVectorizationTaskCommand
2. 根据 imageId 查询 ImageAssetDTO
3. 图片不存在则返回失败
4. 使用 ImageAssetStateChecker.canStartVectorization 校验状态
5. 状态不允许则跳过
6. 构建 ImageVectorizationContext
7. markVectorProcessing
8. 检查 storagePath 文件是否存在
9. 文件不存在则 FailureHandler.handleFileMissing
10. FileStorageService.read 读取图片
11. ActiveCollectionResolver.resolveActiveCollection 获取 active collection
12. 构建 ImageVectorizeRequest
13. ModelVectorizationClient.vectorizeImage
14. 校验 VectorizeResult.embedding 非空和 dim
15. 校验 dim == VectorCollectionConfig.vectorDim
16. 构建 VectorUpsertRequest
17. VectorIndexClient.upsert
18. upsert 成功后 markVectorReady
19. 返回 ImageVectorizationResult(success=true)
```

失败流程：

```text
图片不存在：
返回失败，不重试

状态不允许：
跳过，不作为系统异常

PROCESSING 更新失败：
跳过，避免并发重复任务覆盖状态

文件不存在：
markInvalid + markVectorFailed(FILE_NOT_FOUND)

模型服务失败：
构建 VectorizationFailureContext
交给 RetryPolicy + FailureHandler

向量维度不匹配：
不可重试，markVectorFailed(VECTOR_DIM_MISMATCH)

向量库 upsert 失败：
可重试，未超限回退 PENDING，超限 FAILED

READY 更新失败：
记录 READY_UPDATE_FAILED 事件
依赖 upsert 幂等，等待后续重试或补偿
```

名词解释：`upsert` 是“存在则更新，不存在则插入”。它适合异步任务重试，因为同一个 `imageId` 多次写入不会生成多条向量。

## 9. 最终执行结论

本轮真正要优先完成的是：

```text
ThreadPoolVectorizationTaskPublisher
DefaultImageVectorizationProcessor
DefaultImageAssetQueryService
DefaultImageAssetStatusService
LocalFileStorageService.exists/read
HttpModelVectorizationClient
DefaultVectorIndexClient.upsert
ActiveCollectionResolver
VectorCollectionConfigProvider
VectorizationFailureHandler
VectorizationRetryPolicy
```

最短可运行闭环是：

```text
上传成功
→ 发布任务成功
→ 异步任务开始
→ image_asset 查到图片
→ PENDING 改 PROCESSING
→ 读到本地图片
→ Python 返回 embedding
→ 向量库 upsert 成功
→ MySQL 改 READY
```

只要这条链路跑通，就说明第二轮核心任务完成。搜索相关类可以进入下一轮。
